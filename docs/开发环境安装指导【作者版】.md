# 学成在线三代 开发环境安装指导【作者版】

> 适用于腾讯云 **CentOS 7.6 + Docker 26** 环境（4 核 4G，40GB 系统盘）。  
> 本文档基于作者实际部署过程整理，所有命令均已验证通过。

---

## 说在最前

**腾讯云 Docker 镜像源**：腾讯云 CentOS 镜像已预置 Docker 容器镜像加速，**无需手动配置** `registry-mirrors`。拉取镜像时可直接使用默认配置。

## 二、w0、创建数据目录

```bash
sudo mkdir -p /data/docker/{mysql,redis,minio,rabbitmq,elasticsearch}
sudo chown -R $(whoami):$(whoami) /data/docker
```

---

## 三、w1、MySQL

### 3.1 启动容器

```bash
docker run --name xcplus3-mysql --restart=unless-stopped \
  -v /data/docker/mysql:/var/lib/mysql \
  -p 3306:3306 \
  -e MYSQL_ROOT_PASSWORD='你的密码' \
  -d mysql:8.0.29 \
  --default-authentication-plugin=mysql_native_password \
  --character-set-server=utf8mb4
```

### 3.2 验证

**本机连接**（在服务器上执行）：

```bash
docker exec -it xcplus3-mysql mysql -uroot -p'你的密码' -e "SELECT 1;"
```

**远程连接**（在本地或另一台机器执行）：

```bash
mysql -h 你的服务器公网IP -P 3306 -u root -p
```

> 需在腾讯云安全组放行 **3306** 端口。

## 四、w2、Redis

### 4.1 启动容器

```bash
docker run -d --name xcplus3-redis \
  -p 6379:6379 \
  -v /data/docker/redis:/data \
  --restart unless-stopped \
  redis:7.2-alpine \
  redis-server --appendonly yes --requirepass 你的密码
```

### 4.2 验证

**本机连接**（在服务器上执行）：

```bash
docker exec -it xcplus3-redis redis-cli -a 你的密码 ping
# 预期返回 PONG
```

**远程连接**（在本地或另一台机器执行）：

```bash
redis-cli -h 你的服务器公网IP -p 6379 -a 你的密码 ping
# 预期返回 PONG
```

> 需在腾讯云安全组放行 **6379** 端口。

## 五、w3、MinIO

### 5.1 启动容器

```bash
docker run -d --name xcplus3-minio \
  -p 9000:9000 -p 9001:9001 \
  -e MINIO_ROOT_USER=minioadmin \
  -e MINIO_ROOT_PASSWORD=你的密码 \
  -v /data/docker/minio:/data \
  --restart unless-stopped \
  minio/minio server /data --console-address ":9001"
```

### 5.2 验证

浏览器访问 `http://你的服务器公网IP:9001/`，使用 `minioadmin` 和 `你的密码` 登录。

> 需放行 **9000**、**9001** 端口。

## 六、w4、RabbitMQ

### 6.1 启动容器

```bash
docker run -d --name xcplus3-rabbitmq \
  -p 5672:5672 -p 15672:15672 \
  -e RABBITMQ_DEFAULT_USER=admin \
  -e RABBITMQ_DEFAULT_PASS=你的密码 \
  -v /data/docker/rabbitmq:/var/lib/rabbitmq \
  --restart unless-stopped \
  rabbitmq:3.13-management-alpine
```

### 6.2 安装延迟队列插件

**第一步**：下载插件并上传到服务器

从 [GitHub Releases](https://github.com/rabbitmq/rabbitmq-delayed-message-exchange/releases/tag/v3.13.0) 下载 `rabbitmq_delayed_message_exchange-3.13.0.ez`，通过 FinalShell 等工具上传到服务器 `/tmp` 目录。

**第二步**：拷贝到容器并启用

```bash
docker cp /tmp/rabbitmq_delayed_message_exchange-3.13.0.ez xcplus3-rabbitmq:/plugins

# 进入容器
docker exec -it xcplus3-rabbitmq /bin/sh

# 容器内执行
cd plugins
ls -l | grep delay

# 启用插件
rabbitmq-plugins enable rabbitmq_delayed_message_exchange

# 退出
exit
```

**第三步**：重启 RabbitMQ

```bash
docker restart xcplus3-rabbitmq
```

### 6.3 验证

**方式一**：浏览器访问 `http://你的服务器公网IP:15672`，登录后进入 **Exchanges** 标签：

1. 点击 Exchanges
2. 点击 Add a new exchange
3. 在 Type 下拉框中若有 `x-delayed-message`，说明插件已生效

**方式二**：命令行验证

```bash
docker exec xcplus3-rabbitmq rabbitmq-plugins list | grep delay
# 预期输出：[E*] rabbitmq_delayed_message_exchange 3.13.0
```

> 需放行 **15672**、**25672**、**5671**、**5672** 端口。

## 七、w5、Elasticsearch

### 7.1 启动容器

```bash
docker run -d --name xcplus3-elasticsearch \
  -p 9200:9200 -p 9300:9300 \
  -e discovery.type=single-node \
  -e xpack.security.enabled=false \
  -e "ES_JAVA_OPTS=-Xms512m -Xmx512m" \
  -v /data/docker/elasticsearch:/usr/share/elasticsearch/data \
  --restart unless-stopped \
  elasticsearch:8.11.0
```

### 7.2 常见问题与处理

#### 问题 1：容器反复重启（Crash Loop）

**现象**：`Restarting (1) 48 seconds ago`，容器启动后很快退出。

**排查**：查看日志

```bash
docker logs xcplus3-elasticsearch --tail 100
```

**处理**：设置 `vm.max_map_count`（常见原因）

```bash
sudo sysctl -w vm.max_map_count=262144
echo "vm.max_map_count=262144" | sudo tee -a /etc/sysctl.conf
docker restart xcplus3-elasticsearch
```

等待 1～2 分钟后检查：`docker ps -a | grep elasticsearch`，若状态为 `Up X minutes` 则已正常。

---

#### 问题 2：数据目录权限错误

**现象**：日志中出现 `AccessDeniedException: /usr/share/elasticsearch/data/node.lock`，表示无法写入数据目录。

**处理**：

```bash
# 1. 停止容器
docker stop xcplus3-elasticsearch

# 2. 清理可能损坏的旧数据（首次或异常退出后建议执行）
sudo rm -rf /data/docker/elasticsearch/*

# 3. 设置目录权限（容器内 ES 以 UID 1000 运行）
sudo chown -R 1000:1000 /data/docker/elasticsearch

# 4. 设置 vm.max_map_count（建议保留）
sudo sysctl -w vm.max_map_count=262144

# 5. 启动容器
docker start xcplus3-elasticsearch
```

---

#### 问题 3：验证启动成功

等待 1～2 分钟后执行：

```bash
docker ps | grep elasticsearch
curl http://127.0.0.1:9200
```

若状态为 `Up` 且 `curl` 返回正常 JSON，说明已启动成功。

## 八、w6、Nacos

### 8.1 问题说明

Nacos 2.4.0 开启鉴权时需注意：

1. **环境变量名**：Docker 中应使用 `NACOS_AUTH_TOKEN`，而非 `NACOS_AUTH_TOKEN_SECRET_KEY`
2. **密钥格式**：必须使用 **Base64** 编码（`openssl rand -base64 32`），不能使用 hex

### 8.2 启动容器

```bash
# 1. 停止并删除旧容器（若存在）
docker stop xcplus3-nacos 2>/dev/null; docker rm xcplus3-nacos 2>/dev/null

# 2. 生成 Base64 密钥并启动
KEY=$(openssl rand -base64 32)
docker run -d \
  -e MODE=standalone \
  -e NACOS_AUTH_ENABLE=true \
  -e "NACOS_AUTH_TOKEN=$KEY" \
  -e NACOS_AUTH_IDENTITY_KEY=serverIdentity \
  -e NACOS_AUTH_IDENTITY_VALUE=security \
  -p 8848:8848 -p 9848:9848 -p 9849:9849 \
  --name xcplus3-nacos --restart unless-stopped \
  nacos/nacos-server:v2.4.0

# 3. 查看启动日志
docker logs xcplus3-nacos --tail 20
```

### 8.3 首次访问创建管理员

Nacos 2.4.0 已移除默认密码，开启鉴权后首次访问会进入**初始化页面**，需创建管理员账号和密码，之后使用该账号登录。

### 8.4 防火墙

需放行 **8848**、**9848**、**9849** 端口。

## 十、安全建议

1. **修改默认密码**：MySQL、Redis、MinIO、RabbitMQ、Elasticsearch、Nacos 均使用强密码
2. **腾讯云安全组**：仅开放必要端口，建议限制来源 IP
3. **生产环境**：Elasticsearch 建议启用 xpack 安全认证
