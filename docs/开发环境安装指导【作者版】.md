## 前置与说明

### 腾讯云 Docker 镜像源

腾讯云 CentOS 镜像已预置 Docker 容器镜像加速，**无需手动配置** `registry-mirrors`。拉取镜像时可直接使用默认配置。

| 服务 | 端口 | 说明 |
|------|------|------|
| MySQL | 3306 | 数据库 |
| Redis | 6379 | 缓存 |
| MinIO | 9000, 9001 | API、控制台 |
| RabbitMQ | 5672, 5671 | AMQP |
| RabbitMQ | 15672, 25672 | 管理界面、集群 |
| Elasticsearch | 9200, 9300 | HTTP、传输 |
| Nacos | 8848, 9848, 9849 | HTTP、gRPC |

在腾讯云控制台 → 安全组 → 入站规则中按需放行上述端口。

## 方式一、Docker Compose 一键部署

适用于**新建腾讯云服务器**或**无现有容器**的场景。只需 `docker-compose.yml` 和 `.env`，不依赖 Java 项目代码。与方式二择一即可。

### 1.1 准备目录与文件

**方式 A：从 GitHub 克隆（推荐）**

```bash
cd /home
git clone https://github.com/dexterwu29/xuechengPlus3.git xc-plus3
cd xc-plus3
```

**方式 B：单独建目录**

```bash
mkdir -p /home/xc-plus3
cd /home/xc-plus3
# 用 scp、FinalShell 等上传 docker-compose.yml、.env.example 到此目录
```

### 1.2 前置步骤

```bash
# 1. 创建数据目录
sudo mkdir -p /data/docker/{mysql,redis,minio,rabbitmq,elasticsearch}
sudo chown -R $(whoami):$(whoami) /data/docker

# 2. Elasticsearch 前置（避免 Crash Loop 与权限错误）
sudo sysctl -w vm.max_map_count=262144
echo "vm.max_map_count=262144" | sudo tee -a /etc/sysctl.conf
sudo chown -R 1000:1000 /data/docker/elasticsearch
```

### 1.3 配置环境变量

```bash
cd /home/xc-plus3   # 或项目克隆目录

# 生成 Nacos 鉴权 Token
openssl rand -base64 32
# 将输出复制，稍后填入 .env 的 NACOS_AUTH_TOKEN

# 复制模板并编辑
cp .env.example .env
vim .env   # 或 nano .env
```

`.env` 需填入：

| 变量 | 说明 |
|------|------|
| MYSQL_ROOT_PASSWORD | MySQL root 密码 |
| REDIS_PASSWORD | Redis 密码 |
| MINIO_PASSWORD | MinIO 密码 |
| RABBITMQ_PASSWORD | RabbitMQ 密码 |
| NACOS_AUTH_TOKEN | 上一步 `openssl rand -base64 32` 的输出 |

### 1.4 一键启动

```bash
docker compose up -d
docker compose ps
```

### 1.5 启动后操作

| 服务 | 操作 |
|------|------|
| Nacos | 首次访问 http://服务器IP:8848/nacos 创建管理员账号 |
| RabbitMQ | 安装延迟队列插件（见下方） |

**RabbitMQ 延迟队列插件**：从 [GitHub Releases](https://github.com/rabbitmq/rabbitmq-delayed-message-exchange/releases/tag/v3.13.0) 下载 `rabbitmq_delayed_message_exchange-3.13.0.ez` 并上传到服务器 `/tmp`，然后执行：

```bash
docker cp /tmp/rabbitmq_delayed_message_exchange-3.13.0.ez xcplus3-rabbitmq:/plugins
docker exec -it xcplus3-rabbitmq rabbitmq-plugins enable rabbitmq_delayed_message_exchange
docker restart xcplus3-rabbitmq
```

### 1.6 目录说明

| 目录 | 作用 |
|------|------|
| `/home/xc-plus3` | 放 `docker-compose.yml`、`.env`，执行 `docker compose` 的目录 |
| `/data/docker/...` | 各服务数据持久化目录（由 compose 自动挂载） |

## 方式二、分步部署（docker run）

按组件逐个执行 `docker run`，便于理解与排错。与方式一择一即可。

### 创建数据目录

```bash
sudo mkdir -p /data/docker/{mysql,redis,minio,rabbitmq,elasticsearch}
sudo chown -R $(whoami):$(whoami) /data/docker
```

### MySQL

```bash
docker run --name xcplus3-mysql --restart=unless-stopped \
  -v /data/docker/mysql:/var/lib/mysql \
  -p 3306:3306 \
  -e MYSQL_ROOT_PASSWORD='你的密码' \
  -d mysql:8.0.29 \
  --default-authentication-plugin=mysql_native_password \
  --character-set-server=utf8mb4
```

### Redis

```bash
docker run -d --name xcplus3-redis \
  -p 6379:6379 \
  -v /data/docker/redis:/data \
  --restart unless-stopped \
  redis:7.2-alpine \
  redis-server --appendonly yes --requirepass 你的密码
```

### MinIO

```bash
docker run -d --name xcplus3-minio \
  -p 9000:9000 -p 9001:9001 \
  -e MINIO_ROOT_USER=minioadmin \
  -e MINIO_ROOT_PASSWORD=你的密码 \
  -v /data/docker/minio:/data \
  --restart unless-stopped \
  minio/minio server /data --console-address ":9001"
```

### RabbitMQ

```bash
docker run -d --name xcplus3-rabbitmq \
  -p 5672:5672 -p 15672:15672 \
  -e RABBITMQ_DEFAULT_USER=admin \
  -e RABBITMQ_DEFAULT_PASS=你的密码 \
  -v /data/docker/rabbitmq:/var/lib/rabbitmq \
  --restart unless-stopped \
  rabbitmq:3.13-management-alpine
```

延迟队列插件安装见 [1.5 启动后操作](#15-启动后操作)。

### Elasticsearch

```bash
docker run -d --name xcplus3-elasticsearch \
  -p 9200:9200 -p 9300:9300 \
  -e discovery.type=single-node \
  -e xpack.security.enabled=false \
  -e "ES_JAVA_OPTS=-Xms512m -Xmx512m" \
  -v /data/docker/elasticsearch:/usr/share/elasticsearch/data \
  --restart unless-stopped \
  elasticsearch:8.11.0
```

**常见问题**：

- **Crash Loop**：执行 `sudo sysctl -w vm.max_map_count=262144` 并 `docker restart xcplus3-elasticsearch`
- **数据目录权限**：`sudo chown -R 1000:1000 /data/docker/elasticsearch`

### Nacos

```bash
KEY=$(openssl rand -base64 32)
docker run -d \
  -e MODE=standalone \
  -e NACOS_AUTH_ENABLE=true \
  -e "NACOS_AUTH_TOKEN=$KEY" \
  -e NACOS_AUTH_IDENTITY_KEY=serverIdentity \
  -e NACOS_AUTH_IDENTITY_VALUE=security \
  -p 8848:8848 -p 9848:9848 -p 9849:9849 \
  --name xcplus3-nacos --restart unless-stopped \
  nacos/nacos-server:v2.4.0
```

> Nacos 2.4.0 开启鉴权后首次访问需创建管理员账号。参考：[用户手册 - 配置鉴权](https://nacos.io/docs/next/manual/user/auth/) | [运维手册 - 权限校验](https://nacos.io/docs/next/manual/admin/auth/)

## 附录

### A. 安全建议

1. **修改默认密码**：各组件均使用强密码
2. **腾讯云安全组**：仅开放必要端口，建议限制来源 IP
3. **生产环境**：Elasticsearch 建议启用 xpack 安全认证

### B. 数据目录约定

| 服务 | 挂载路径 |
|------|----------|
| MySQL | /data/docker/mysql |
| Redis | /data/docker/redis |
| MinIO | /data/docker/minio |
| RabbitMQ | /data/docker/rabbitmq |
| Elasticsearch | /data/docker/elasticsearch |

### C. MySQL 旧数据与密码

若 `/data/docker/mysql` 已有之前部署的数据，MySQL 会沿用**旧数据里的 root 密码**，不会使用 `.env` 中的 `MYSQL_ROOT_PASSWORD`。

**处理方式**：

- 用之前设置的 MySQL root 密码连接
- 或清空数据目录重新初始化（**会丢失所有数据库**）：

```bash
docker stop xcplus3-mysql
docker rm xcplus3-mysql
sudo mv /data/docker/mysql /data/docker/mysql.bak
sudo mkdir -p /data/docker/mysql
sudo chown -R 999:999 /data/docker/mysql
docker compose up -d xcplus3-mysql
```

### D. 从分步部署切换到一键部署

若当前为分步部署的容器，想切换到 Docker Compose：

1. **拍快照**（可选）
2. **记录各服务密码**，填入 `.env`
3. **一键停止并删除容器**：

```bash
for c in xcplus3-mysql xcplus3-redis xcplus3-minio xcplus3-rabbitmq xcplus3-elasticsearch xcplus3-nacos; do docker stop $c 2>/dev/null; docker rm $c 2>/dev/null; done
```

4. 进入项目目录，按 [方式一、Docker Compose 一键部署](#方式一docker-compose-一键部署) 执行 `docker compose up -d`
