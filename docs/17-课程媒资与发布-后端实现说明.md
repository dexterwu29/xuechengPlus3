# 课程媒资与发布 - 后端实现说明

## 一、已实现功能

### 1. 媒资上传（MD5 去重 + 断点续传）

| 接口 | 方法 | 说明 |
|------|------|------|
| `/content/media/upload/check` | POST | 检查 MD5 是否已存在，存在返回 fileId |
| `/content/media/upload` | POST | 小文件直传（<5MB），支持 MD5 去重 |
| `/content/media/upload/init` | POST | 初始化分片上传 |
| `/content/media/upload/status/{uploadId}` | GET | 查询已上传分片索引 |
| `/content/media/upload/chunk/{uploadId}/{chunkIndex}` | POST | 上传单个分片 |
| `/content/media/upload/complete/{uploadId}` | POST | 合并分片并落库 |
| `/content/media/{fileId}` | GET | 获取文件信息（含预签名 URL） |

**请求头**：所有 `/media/**` 需携带 `X-Company-Id`。

### 2. 课程图片

| 接口 | 方法 | 说明 |
|------|------|------|
| `/content/courses/{id}/pic` | PUT | 更新课程封面，参数 `fileId` |

### 3. 教学计划媒资（一节多文件）

| 接口 | 方法 | 说明 |
|------|------|------|
| `/content/teachplans/{teachplanId}/media` | POST | 绑定媒资（可多次调用，一节多文件） |
| `/content/teachplans/{teachplanId}/media` | GET | 查询该节媒资列表 |
| `/content/teachplans/{teachplanId}/media` | DELETE | 解绑该节全部媒资 |
| `/content/teachplans/{teachplanId}/media/{fileId}` | DELETE | 解绑指定媒资 |

**绑定请求体**：`{ "fileId": "xxx", "mediaFileName": "视频.mp4", "mediaType": "video", "orderBy": 0 }`  
`mediaId` 与 `fileId` 二选一，`mediaType` 默认 `video`，`orderBy` 默认 0。

### 4. 课程发布

| 接口 | 方法 | 说明 |
|------|------|------|
| `/content/courses/{id}/publish` | POST | 发布课程（publish_status → 203002） |

### 5. 课程预览（Thymeleaf）

| 接口 | 方法 | 说明 |
|------|------|------|
| `/content/courses/{id}/preview` | GET | 机构内部预览（需 X-Company-Id） |
| `/content/public/courses/{id}/preview` | GET | 公开预览（已发布课程，无需 X-Company-Id） |

### 6. 公开接口（无需 X-Company-Id）

| 接口 | 方法 | 说明 |
|------|------|------|
| `/content/public/courses` | GET | 已发布课程分页，参数 pageNo、pageSize、courseName、mt、st |
| `/content/public/courses/{id}` | GET | 已发布课程详情 |
| `/content/public/courses/{id}/teachplans` | GET | 已发布课程教学计划树 |

## 二、数据库迁移

执行顺序：

1. `initSql/xcplus_content.sql`（若未执行）
2. `initSql/xcplus_content_media.sql`

`xcplus_content_media.sql` 会：

- 创建 `media_file`、`upload_task`、`upload_chunk` 表
- 为 `teachplan_media` 增加 `file_id`、`media_type`、`order_by` 列

若 `teachplan_media` 已存在这些列，ALTER 会报错，需手动跳过或调整脚本。

---

## 三、配置

在 `application-local.yml` 中配置 MinIO（腾讯云）：

```yaml
minio:
  endpoint: http://<MinIO地址>:9000
  access-key: <accessKey>
  secret-key: <secretKey>
  bucket: xcplus
  chunk-size: 5242880  # 5MB
```

若 bucket 不存在，启动时会自动创建。

---

## 四、课程封面 pic 字段约定

- 更新封面时传入 `fileId`，后端存储为 `media:{fileId}`
- 前端展示时：若 `pic` 以 `media:` 开头，可调用 `GET /content/media/{fileId}` 获取预签名 URL 用于展示
