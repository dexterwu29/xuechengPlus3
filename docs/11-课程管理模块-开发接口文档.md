## 一、端口与运行环境规划

### 1.1 微服务端口分配

| 端口 | 微服务 | 说明 |
|------|--------|------|
| 11001 | xc-plus-content-service | 课程管理（Content） |
| 12001 | xc-plus-media-service | 媒资管理 |
| 13001 | xc-plus-system-service | 系统管理（字典等） |
| 14001 | xc-plus-auth-service | 认证服务 |
| 15001 | xc-plus-learning-service | 学习服务 |
| 63010 | xc-plus-gateway | API 网关（预留） |

### 1.2 开发环境约定

- **后端服务**：本地启动，如 `localhost:11001`
- **前端**：本地启动，如 `localhost:5173`（Vite 默认）
- **Nacos**：腾讯云（必须）
- **MySQL**：腾讯云（必须），微服务拆分后各服务独立建库
- **Redis / MinIO / RabbitMQ 等**：腾讯云（必须）

### 1.3 数据库与微服务对应

| 库名 | 所属微服务 |
|------|------------|
| xcplus_content | xc-plus-content-service |
| xcplus_system | xc-plus-system-service |
| xcplus_media | xc-plus-media-service（后续） |
| 其他 | 按模块独立建库 |

### 1.4 接口访问路径约定

- 直连微服务：`http://localhost:11001/content/...`
- 经网关（后续）：`http://<gateway-host>:63010/content/...`

## 二、命名规范

### 2.1 数据库字段（下划线）

- 多词连接：`xx_id`、`teach_mode`、`media_file_name`、`order_by`
- 时间：`create_time`、`update_time`
- 操作人：`create_by`、`update_by`
- 布尔/状态：`is_xxx`，后缀用形容词或动词被动，语法成立。例如：
  - `is_visible`（是否可见）
  - `is_leaf`（是否为叶子节点，树结构惯例）
  - `is_preview_enabled`（是否可试看）
  - `is_deleted`（是否已删除，逻辑删除）

### 2.2 Java 实体类（驼峰）

- 数据库 `parent_id` → Java `parentId`
- 数据库 `create_time` → Java `createTime`
- 数据库 `media_file_name` → Java `mediaFileName`

### 2.3 描述类字段

- 课程介绍、教师简介等统一使用 `description`

## 五、RESTful 接口定义

### 5.1 统一规范

- **Content-Type**：`application/json`
- **基础路径**：`/content`
- **分页**：`pageNo`、`pageSize` 作为 Query 参数
- **统一响应**：`{ "code": 0, "msg": "success", "data": {...} }`
- **错误码**：4xx 业务错误，5xx 系统错误
- **单复数**：资源路径用复数（courses、teachers、teachplans），单资源用 id（courses/{id}）

### 5.2 课程分类

## 七、越权与权限控制

### 7.1 机构隔离

- **测试机构 ID**：`1`、`2`（用于越权测试）
- 所有课程、教师、计划等操作须校验 `company_id`
- 非本机构数据不可查询、修改、删除

### 7.2 校验规则

| 场景 | 校验 |
|------|------|
| 查询课程 | 仅能查询本机构课程 |
| 修改/删除课程 | 仅能操作本机构课程 |
| 教学计划 | 通过 course_id 间接校验机构 |
| 师资 | 通过 course_id 间接校验机构 |

### 7.3 本阶段实现方式

- 请求头或参数传入 `company_id`（如 `X-Company-Id`），后端校验与数据所属机构一致
- 后续接入登录后，从 Token 解析机构 ID，替换手动传参

## 八、配置文件约定

### 8.1 原则

- 除项目本身在本地启动外，**Nacos、MySQL、Redis 等全部使用腾讯云**
- 提供 `*-local.yml`（不被 Nacos 托管，用户自行补充账号密码）

### 8.2 文件对应

| 类型 | 文件名 | 说明 |
|------|--------|------|
| 本地配置 | application-local.yml | 实际使用，.gitignore 排除 |
| 示例配置 | application-example.yml | 字段与 local 一致，供复制 |

### 8.3 字段一致性

- `application-example.yml` 与 `application-local.yml` **字段必须完全一致**
- 仅将敏感值替换为占位符（如 `your_mysql_password`）
