## 第五：媒资存储路径与 MD5 缓存

### 5.1 当前路径
```
xcplus3-video/1/media/3a9263965948499a.mp4
```

### 5.2 建议路径（MD5 分层，含存储桶）

**完整存储路径格式：**
```
{bucket}/{md5[0]}/{md5[1]}/{md5}/{md5}_{原文件名}.{后缀}
```

**示例：**
- 存储桶：`xcplus3-video`
- 对象路径：`3/a/3a9263965948499a/3a9263965948499a_测试视频.mp4`
- 完整路径：`xcplus3-video/3/a/3a9263965948499a/3a9263965948499a_测试视频.mp4`

**说明：**
- `bucket`：MinIO 存储桶名称，可配置（如 `xcplus3-video`、`xcplus3-doc`）
- `md5[0]`、`md5[1]`：MD5 首两位字符，用于目录分散
- `md5`：完整 MD5 值
- `原文件名`：不包含扩展名，仅用于可读性

**优点：**
- 目录分散，避免单目录文件过多
- 通过 MD5 可快速定位文件
- 支持秒传：DB 有 MD5 且 MinIO 有对应文件 → 直接返回成功

### 5.3 缓存区（秒传）逻辑
1. 用户上传文件，计算 MD5
2. 查询 DB：是否存在相同 MD5 的媒资记录
3. 查询 MinIO：是否存在对应路径的文件
4. 若两者同时满足 → 直接关联，返回成功，无需再传
5. 否则：按新路径上传并存储

### 5.4 实现要点
- 媒资服务：上传接口先查 MD5，命中则秒传
- MinIO 路径：按新规则生成 `objectName`，`bucket` 单独配置

## 第二：课程封面与文件限制

### 2.1 封面规则
- **数量**：最多 3 张图片
- **默认**：可编辑者选择 1 张作为默认展示（课程列表/详情主图）
- **其余**：最多 2 张，仅出现在课程详情页
- **模板**：课程详情页需预留多图展示区域

### 2.2 文件大小限制
- **前端**：上传前校验单张图片大小（建议 ≤ 2MB）
- **后端**：Spring Boot `spring.servlet.multipart.max-file-size`、`max-request-size` 需放开或合理配置
- **检查位置**：`application.yml` / `application-local.yml`

### 2.3 数据模型
- `course_base` 或扩展表需支持多封面字段，如 `cover_pics`（JSON 数组）、`default_cover_index`（默认图索引）

**实现状态：**
| 项 | 状态 | 说明 |
|----|------|------|
| 2.2 后端 multipart | ✅ 已落实 | `application.yml` 已配置 `max-file-size: 100MB`、`max-request-size: 100MB`，支持 47.9MB 等大视频分片上传 |
| 2.2 前端封面上传 ≤2MB | ❌ 未落实 | `CoursePicUpload` 使用 `MediaUpload`，无单图 2MB 校验 |
| 2.1 / 2.3 多封面 | ❌ 未落实 | 当前仅 `pic` 单字段，无 `cover_pics`、`default_cover_index` |

- **QQ**：必填
- **微信**：必填
- **手机号**：必填，且需满足国内 86 号段校验（如 1[3-9]\d{9}）

## 第四：教学计划（教学大纲）页面

### 4.1 布局重设计
**现状问题：**
- 课程章节标题不在一行（width 不足，文字竖排）
- 上传文件区域占用过大

**目标：**
- 按功能合理分布：左侧章节树、右侧媒资/操作区
- 章节标题保证足够宽度，单行显示
- 上传区域可折叠或收缩，避免喧宾夺主

### 4.2 上传文件当页预览
- 视频：支持在当页内嵌播放器预览
- 文档：支持在线预览（如 PDF 预览、或跳转预览页）

### 4.3 试看功能
- 付费课支持将**指定章**或**指定节**设为试看
- 需在 `teachplan` 或 `teachplan_media` 增加 `is_preview` 字段
- 前端：章节/节可勾选「试看」

### 4.4 业务规则
- **教学计划为空**：禁止进入下一步（课程创建/编辑流程）
- **有子节的章**：不允许删除（需先删除所有子节）

**实现状态：**
| 项 | 状态 | 说明 |
|----|------|------|

| 4.4 教学计划为空禁止下一步 | ✅ 已落实 | `goToStep4`、`doSubmit` 均有 `teachplanTree.length` 校验 |
| 4.4 有子节禁止删章 | ❌ 未落实 | 删除章时未校验是否有子节 |
| 4.1 布局重设计 | ❌ 未落实 | 章节树与上传区布局未按需求调整 |
