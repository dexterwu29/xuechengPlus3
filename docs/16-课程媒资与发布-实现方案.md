# 课程媒资与发布 - 实现方案

## 一、功能点概览

| 功能 | 说明 |
|------|------|
| 课程图片上传 | 课程封面图，支持上传到 MinIO |
| 教学大纲媒资 | 章/节可绑定多个视频或文档（Word/PPT/PDF/MD/ZIP 等） |
| MD5 去重 | 全 MinIO 系统内按 MD5 去重，相同文件只存一份 |
| 断点续传 | 视频、ZIP 等大文件（>5MB）支持分片上传、断点续传 |
| 课程预览 | 编辑完成后可预览课程介绍页（模板渲染） |
| 课程发布 | 机构确认后发布，更新 publish_status |
| 首页展示 | 已发布课程在首页展示 |
| 课程详情 | 点击进入课程预览 + 联系销售报名咨询 |

## 二、库表设计

### 2.1 media_file（媒资文件表，MD5 去重）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | bigint | 主键 |
| file_id | varchar(32) | 业务 ID（UUID 短码），对外暴露 |
| md5 | varchar(32) | 文件 MD5，唯一索引 |
| file_path | varchar(500) | MinIO 对象路径 |
| file_name | varchar(255) | 原始文件名 |
| file_size | bigint | 文件大小（字节） |
| file_type | varchar(20) | 类型：video/doc/image/zip |
| content_type | varchar(100) | MIME 类型 |
| bucket | varchar(64) | MinIO bucket |
| company_id | bigint | 机构 ID |
| create_time | datetime | 创建时间 |

### 2.2 upload_task（分片上传任务）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | bigint | 主键 |
| upload_id | varchar(64) | 上传任务 ID |
| file_md5 | varchar(32) | 文件 MD5 |
| file_name | varchar(255) | 文件名 |
| file_size | bigint | 文件大小 |
| chunk_size | int | 分片大小 |
| chunk_count | int | 分片总数 |
| status | tinyint | 0 进行中 1 已完成 |
| company_id | bigint | 机构 ID |
| create_time | datetime | 创建时间 |

### 2.3 upload_chunk（已上传分片）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | bigint | 主键 |
| upload_id | varchar(64) | 任务 ID |
| chunk_index | int | 分片序号 |
| chunk_path | varchar(500) | MinIO 临时路径 |
| create_time | datetime | 上传时间 |

### 2.4 teachplan_media 扩展

现有表已支持一节多文件（多行）。需增加：
- `file_id`：关联 media_file.file_id
- `media_type`：video/doc
- `order_by`：同一节内排序

## 三、技术栈

| 组件 | 选型 |
|------|------|
| 对象存储 | MinIO |
| 模板引擎 | Thymeleaf（课程预览页） |
| 分片上传 | 自定义（chunk 存 MinIO temp，完成后 ComposeObject 合并） |
| 前端上传 | axios + 分片逻辑，支持断点续传 |

## 四、API 设计

### 4.1 媒资上传

- `POST /media/upload/check`：检查 MD5 是否已存在，存在则返回 fileId
- `POST /media/upload`：小文件直传（<5MB）
- `POST /media/upload/init`：初始化分片上传
- `GET /media/upload/status/{uploadId}`：查询已上传分片
- `POST /media/upload/chunk/{uploadId}/{chunkIndex}`：上传分片
- `POST /media/upload/complete/{uploadId}`：合并分片并落库

### 4.2 课程相关

- `PUT /courses/{id}/pic`：更新课程图片（传 fileId 或 URL）
- `GET /media/{fileId}`：获取文件信息/预览 URL
- `POST /courses/{id}/preview`：生成预览页 URL
- `POST /courses/{id}/publish`：发布课程

### 4.3 公开接口（无需 X-Company-Id）

- `GET /public/courses`：已发布课程列表（分页）
- `GET /public/courses/{id}`：课程详情（含介绍、大纲、媒资）
- `GET /public/courses/{id}/preview`：课程预览页 HTML
