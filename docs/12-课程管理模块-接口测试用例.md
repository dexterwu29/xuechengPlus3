## 二、课程管理

### 2.1 POST /content/courses/page 分页查询（Body）

| 用例 | 预期 | 请求 Body |
|------|------|-----------|
| 成功-无条件 | 200，返回分页数据 | `{}` |
| 成功-按名称 | 200，模糊匹配 | 见下 ① |
| 成功-按审核状态 | 200 | 见下 ② |
| 成功-按发布状态 | 200 | 见下 ③ |
| 成功-组合条件 | 200 | 见下 ④ |

① 按名称：
```json
{
  "pageNo": 1,
  "pageSize": 10,
  "courseName": "Java"
}
```

② 按审核状态：
```json
{
  "pageNo": 1,
  "pageSize": 10,
  "auditStatus": "202004"
}
```

③ 按发布状态：
```json
{
  "pageNo": 1,
  "pageSize": 10,
  "publishStatus": "203002"
}
```

④ 组合条件：
```json
{
  "pageNo": 1,
  "pageSize": 5,
  "courseName": "AI",
  "auditStatus": "202004",
  "publishStatus": "203002"
}
```

### 2.2 GET /content/courses 分页查询（Query）

| 用例 | 预期 | Query 参数 |
|------|------|------------|
| 成功 | 200 | `pageNo=1&pageSize=10` |
| 成功-带条件 | 200 | `pageNo=1&pageSize=10&courseName=Python&auditStatus=202004&publishStatus=203002` |

## 七、测试前准备

1. **数据库**：执行 `xcplus_content.sql` 初始化（含 22 门课程，丰富 auditStatus/publishStatus）
2. **服务**：启动 xc-plus-content-service，端口 11001
3. **请求头**：除 `/content/course-categories/tree` 外，均需 `X-Company-Id: 1` 或 `2`
4. **机构归属**：课程 1–15 属机构 1，课程 16–22 属机构 2

## 八、curl 示例（复制即用，含成功/失败用例）

> 推荐使用 IDEA 打开 `xc-plus-content-service/http/课程管理接口测试.http` 点击 **Run all requests** 一键执行；或运行 `./http/run-tests.sh` 收集结果到 `http/results/`。

### 成功用例

```bash
# 1. 获取分类树（无需 X-Company-Id）
curl -X GET "http://localhost:11001/content/course-categories/tree"

# 2. 分页-无条件
curl -X POST "http://localhost:11001/content/courses/page" \
  -H "Content-Type: application/json" -H "X-Company-Id: 1" \
  -d '{"pageNo":1,"pageSize":10}'

# 3. 分页-按名称
curl -X POST "http://localhost:11001/content/courses/page" \
  -H "Content-Type: application/json" -H "X-Company-Id: 1" \
  -d '{"pageNo":1,"pageSize":10,"courseName":"Java"}'

# 4. 分页-按审核状态（验证 total 正确）
curl -X POST "http://localhost:11001/content/courses/page" \
  -H "Content-Type: application/json" -H "X-Company-Id: 1" \
  -d '{"pageNo":1,"pageSize":10,"auditStatus":"202004"}'

# 5. 分页-按发布状态 pageSize=2（验证分页生效）
curl -X POST "http://localhost:11001/content/courses/page" \
  -H "Content-Type: application/json" -H "X-Company-Id: 1" \
  -d '{"pageNo":1,"pageSize":2,"publishStatus":"203002"}'

# 6. 新增课程
curl -X POST "http://localhost:11001/content/courses" \
  -H "Content-Type: application/json" -H "X-Company-Id: 1" \
  -d '{"name":"接口测试新增课程","users":"测试学员","tags":"测试","mt":"1-001-001","st":"1-001-001","grade":"204001","teachMode":"200001","description":"用于接口测试"}'

# 7. 获取课程详情
curl -X GET "http://localhost:11001/content/courses/1" -H "X-Company-Id: 1"

# 8. 更新课程
curl -X PUT "http://localhost:11001/content/courses/1" \
  -H "Content-Type: application/json" -H "X-Company-Id: 1" \
  -d '{"name":"Java核心技术（已更新）","users":"零基础","tags":"Java","mt":"1-001-001","st":"1-001-001","grade":"204001","teachMode":"200001","description":"更新"}'

# 9. 删除课程（逻辑删除，机构2的课程22）
curl -X DELETE "http://localhost:11001/content/courses/22" -H "X-Company-Id: 2"

# 10. 获取教学计划树
curl -X GET "http://localhost:11001/content/courses/1/teachplans" -H "X-Company-Id: 1"

# 11. 新增教学计划
curl -X POST "http://localhost:11001/content/courses/1/teachplans" \
  -H "Content-Type: application/json" -H "X-Company-Id: 1" \
  -d '{"pname":"第4章 测试章","parentId":0,"grade":1,"orderBy":4,"isPreviewEnabled":"0"}'

# 12. 获取/保存营销
curl -X GET "http://localhost:11001/content/courses/1/market" -H "X-Company-Id: 1"
curl -X PUT "http://localhost:11001/content/courses/1/market" \
  -H "Content-Type: application/json" -H "X-Company-Id: 1" \
  -d '{"charge":"201001","price":199,"originalPrice":299,"validDays":365}'

# 13. 修改/删除教学计划、上移、下移
curl -X PUT "http://localhost:11001/content/teachplans/1" \
  -H "Content-Type: application/json" -H "X-Company-Id: 1" \
  -d '{"pname":"第1章 Java基础（已修改）","parentId":0,"grade":1,"orderBy":1,"isPreviewEnabled":"0"}'
curl -X DELETE "http://localhost:11001/content/teachplans/8" -H "X-Company-Id: 1"
curl -X POST "http://localhost:11001/content/teachplans/5/move-up" -H "X-Company-Id: 1"
curl -X POST "http://localhost:11001/content/teachplans/1/move-down" -H "X-Company-Id: 1"

# 14. 绑定/解绑媒资
curl -X POST "http://localhost:11001/content/teachplans/4/media" \
  -H "Content-Type: application/json" -H "X-Company-Id: 1" \
  -d '{"mediaId":"3a5a861d1c745d05166132c47b44f9e4","mediaFileName":"01-Nacos.avi"}'
curl -X DELETE "http://localhost:11001/content/teachplans/4/media" -H "X-Company-Id: 1"

# 15. 师资 CRUD
curl -X GET "http://localhost:11001/content/courses/1/teachers" -H "X-Company-Id: 1"
curl -X POST "http://localhost:11001/content/courses/1/teachers" \
  -H "Content-Type: application/json" -H "X-Company-Id: 1" \
  -d '{"teacherName":"测试讲师","position":"初级讲师","description":"接口测试"}'
curl -X PUT "http://localhost:11001/content/courses/1/teachers/1" \
  -H "Content-Type: application/json" -H "X-Company-Id: 1" \
  -d '{"teacherName":"张老师（已更新）","position":"高级讲师","description":"已更新"}'
curl -X DELETE "http://localhost:11001/content/courses/1/teachers/2" -H "X-Company-Id: 1"
```

### 失败用例（预期 4xx/5xx）

```bash
# 新增课程-名称为空（预期 400/500）
curl -X POST "http://localhost:11001/content/courses" \
  -H "Content-Type: application/json" -H "X-Company-Id: 1" \
  -d '{"name":"","users":"测试","tags":"测试","mt":"1-001-001","st":"1-001-001","grade":"204001","teachMode":"200001","description":"应失败"}'

# 新增课程-必填缺失（预期 400/500）
curl -X POST "http://localhost:11001/content/courses" \
  -H "Content-Type: application/json" -H "X-Company-Id: 1" \
  -d '{"users":"测试","tags":"测试","mt":"1-001-001","st":"1-001-001","grade":"204001","teachMode":"200001"}'

# 获取详情-不存在（预期 404）
curl -X GET "http://localhost:11001/content/courses/99999" -H "X-Company-Id: 1"

# 获取详情-越权（课程16属机构2，预期 403）
curl -X GET "http://localhost:11001/content/courses/16" -H "X-Company-Id: 1"

# 删除-越权（课程1属机构1，预期 403）
curl -X DELETE "http://localhost:11001/content/courses/1" -H "X-Company-Id: 2"

# 保存营销-价格非法（预期 400/500）
curl -X PUT "http://localhost:11001/content/courses/1/market" \
  -H "Content-Type: application/json" -H "X-Company-Id: 1" \
  -d '{"charge":"201001","price":-100,"originalPrice":299,"validDays":365}'
```
