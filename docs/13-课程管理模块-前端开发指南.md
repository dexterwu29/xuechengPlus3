## 一、概述

### 1.1 模块定位

课程管理模块（Content）是学成在线三代的核心业务模块之一，负责课程基本信息、分类、营销、教学计划、师资、媒资关联等全生命周期管理。前端需与 `xc-plus-content-service`（端口 11001）对接。

### 1.2 技术栈约定

| 层级 | 技术选型 |
|------|----------|
| 前端 | Vue 3、Node.js 20+、Vite 5.x |
| 后端 | Spring Boot 3.4+、RESTful JSON |
| 接口风格 | 统一响应 `{ code, msg, data }`，分页 `{ items, total, pageNo, pageSize }` |

### 1.3 架构示意

```
Vue 3 前端 ──► API 网关（预留） ──► xc-plus-content-service (11001)
                    │
                    └──► Nacos / MySQL / Redis（腾讯云）
```

**本阶段**：前端直连 `http://localhost:11001/content`，不经过网关。

## 二、环境与前置

### 2.1 后端服务启动

```bash
cd xc-plus3/backend/xc-plus-content-service
mvn spring-boot:run -Dspring-boot.run.profiles=local
```

- 健康检查：`http://localhost:11001/content/actuator/health`
- Knife4j 文档：`http://localhost:11001/content/doc.html`
- OpenAPI：`http://localhost:11001/content/v3/api-docs`

### 2.2 机构隔离（必读）

除 `/content/course-categories/tree` 外，**所有课程相关接口**必须在请求头中传入机构 ID：

```
X-Company-Id: 1
```

或

```
X-Company-Id: 2
```

- 测试机构 ID：`1`、`2`（用于越权测试）
- 后续接入登录后，从 Token 解析机构 ID，替换手动传参

## 三、API 基础配置

### 3.1 基础地址

| 环境 | 基础地址 |
|------|----------|
| 本地开发 | `http://localhost:11001` |
| Content 模块前缀 | `/content` |
| 完整基础路径 | `http://localhost:11001/content` |

### 3.2 统一响应结构

```typescript
interface ApiResponse<T> {
  code: number;   // 0 表示成功
  msg: string;
  data: T | null;
}

interface PageResult<T> {
  items: T[];
  total: number;
  pageNo: number;
  pageSize: number;
}
```

### 3.3 错误处理

- `code === 0`：成功
- `code !== 0`：业务错误或系统错误，`msg` 为错误描述
- 4xx：业务错误（如 403 越权、404 资源不存在）
- 5xx：系统错误

## 四、接口清单与数据模型

### 4.1 课程分类

| 方法 | 路径 | 说明 | 需 X-Company-Id |
|------|------|------|-----------------|
| GET | /content/course-categories/tree | 获取分类树 | 否 |

**响应**：`ApiResponse<CourseCategoryTreeVO[]>`，根节点 id 为 `"1"`。

```typescript

  id: string;
  name: string;
  label: string;
  parentId: string;

  orderBy?: number;
  isLeaf?: number;

}
```

### 4.2 课程管理

| 方法 | 路径 | 说明 | 需 X-Company-Id |
|------|------|------|-----------------|
| POST | /content/courses/page | 分页查询（Body） | 是 |
| GET | /content/courses | 分页查询（Query） | 是 |
| POST | /content/courses | 新增课程 | 是 |
| GET | /content/courses/{id} | 获取课程详情 | 是 |
| PUT | /content/courses/{id} | 更新课程 | 是 |
| DELETE | /content/courses/{id} | 逻辑删除课程 | 是 |

**分页查询请求体**（POST /courses/page）：

```typescript
interface CoursePageQueryDTO {
  pageNo?: number;      // 默认 1
  pageSize?: number;    // 默认 10
  courseName?: string;  // 模糊
  auditStatus?: string; // 202001/202002/202003/202004
  publishStatus?: string; // 203001/203002/203003
}
```

**新增/更新课程请求体**：

```typescript
interface CourseCreateDTO {
  name: string;        // 必填
  users?: string;
  tags?: string;
  mt?: string;         // 大分类ID
  st?: string;         // 小分类ID
  grade?: string;      // 204001/204002/204003
  teachMode?: string;  // 200001/200002/200003
  description?: string;
  pic?: string;
}
```

**课程列表项**：

```typescript
interface CourseBaseVO {
  id: number;
  companyId: number;
  companyName: string;
  name: string;
  users?: string;
  tags?: string;
  mt?: string;
  st?: string;
  grade?: string;
  teachMode?: string;
  description?: string;
  pic?: string;
  createTime: string;  // ISO 8601
  updateTime: string;
  auditStatus: string;
  publishStatus: string;
}
```

**课程详情**（含营销）：`CourseDetailVO` 在 `CourseBaseVO` 基础上增加 `charge`、`price`、`originalPrice`、`qq`、`wechat`、`phone`、`validDays`。

```
└── ...
```

### 6.2 请求封装示例（axios）

```typescript
// api/request.ts
import axios from 'axios';

const request = axios.create({
  baseURL: 'http://localhost:11001/content',
  timeout: 10000,
  headers: { 'Content-Type': 'application/json' },
});

// 请求拦截：注入 X-Company-Id（后续从 Token 解析）
request.interceptors.request.use((config) => {
  const companyId = localStorage.getItem('companyId') || '1';
  config.headers['X-Company-Id'] = companyId;
  return config;
});

// 响应拦截：统一处理 code !== 0
request.interceptors.response.use(
  (res) => {
    const { code, msg, data } = res.data;
    if (code !== 0) {
      return Promise.reject(new Error(msg || '请求失败'));
    }
    return data; // 直接返回 data，便于业务层使用
  },
  (err) => Promise.reject(err)
);

export default request;
```

### 6.3 API 方法示例

```typescript
// api/content/category.ts
import request from '../request';

export function getCategoryTree() {
  return request.get<CourseCategoryTreeVO[]>('/course-categories/tree');
}

// api/content/course.ts
import request from '../request';

export function pageCourses(params: CoursePageQueryDTO) {
  return request.post<PageResult<CourseBaseVO>>('/courses/page', params);
}

export function getCourse(id: number) {
  return request.get<CourseDetailVO>(`/courses/${id}`);
}

export function createCourse(data: CourseCreateDTO) {
  return request.post<number>('/courses', data);
}

export function updateCourse(id: number, data: CourseCreateDTO) {
  return request.put(`/courses/${id}`, data);
}

export function deleteCourse(id: number) {
  return request.delete(`/courses/${id}`);
}
```

## 七、开发顺序建议

1. **Phase 1**：分类树、课程 CRUD、课程营销、逻辑删除
2. **Phase 2**：教学计划树、教学计划 CRUD、上移下移、逻辑删除
3. **Phase 3**：师资管理（含单教师查询）、课程逻辑删除（级联）
4. **Phase 4**：教学计划-媒资关联（依赖媒资服务 12001）
5. **Phase 5**：课程发布、联调与验收

## 八、常见问题与注意事项

### 8.1 必须传 X-Company-Id

除分类树外，所有接口均需 `X-Company-Id`，否则会返回 403 或业务错误。

### 8.2 分页查询方式

- **推荐**：`POST /courses/page`，Body 为 `CoursePageQueryDTO`，支持条件 + 分页
- **可选**：`GET /courses?pageNo=1&pageSize=10&courseName=java`，仅支持简单条件

### 8.3 分类树 children 字段名

后端返回子节点字段为 `childrenTreeNodes`，前端可统一映射为 `children` 便于树组件使用。

### 8.4 教学计划层级

- `grade === 1`：章
- `grade === 2`：节
- `parentId === 0`：顶级章

### 8.5 逻辑删除

课程、教师、计划等均为逻辑删除，前端删除后列表不再展示，但数据库仍保留。

### 8.6 越权防护

后端校验机构归属，跨机构访问会返回 403。前端应避免暴露其他机构 ID 的入口。

## 十、自动化测试

后端提供 Python 自动化测试脚本，可用于验证接口是否正常：

```bash
cd xc-plus3/backend/xc-plus-content-service/http
```
