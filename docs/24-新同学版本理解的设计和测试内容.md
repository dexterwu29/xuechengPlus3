## 一、核心设计理念

### 1.1 角色定义（简化版）

| 角色 | 说明 | 职责范围 |
|------|------|----------|
| **visitor** | 访客/学员 | 浏览已发布课程，登录后可观看免费课/试看章节，付费后可观看收费课 |
| **teacher** | 机构教师 | 隶属于特定机构，负责创建课程、上传媒资，**只能管理本机构内容** |
| **super_admin** | 超级管理员 | 系统级管理员，**审核所有机构提交的课程**，具有最高审核权限 |

> **重要变更**：移除原 `admin` 角色，不再有单独的机构管理员

### 1.2 核心业务原则

1. **机构隔离**：teacher 只能 CRUD 本机构的课程，看不到其他机构的编辑入口
2. **审核集中**：所有机构的课程审核由 super_admin 统一处理
3. **站内信通知**：关键状态变更必须通过站内信通知
4. **权限分层**：课程信息（A）与媒资访问（B）分离

## 二、课程状态流转设计

### 2.1 状态码定义（三位）

| 位 | 含义 | 取值 |
|----|------|------|
| 第1位 | 提交状态 | 0=未提交，1=已提交 |
| 第2位 | 审核状态 | 0=待审核，1=已审核 |
| 第3位 | 发布状态 | 0=未发布，1=已发布，2=已下架 |

### 2.2 状态组合

| 状态码 | 提交 | 审核 | 发布 | 名称 | 说明 |
|--------|------|------|------|------|------|
| **000** | 0 | 0 | 0 | 草稿 | 未提交，待审核，未发布 |
| **100** | 1 | 0 | 1 | 待审核 | 已提交，待审核，未发布 |
| **111** | 1 | 1 | 1 | 已发布 | 已提交，已审核，已发布 |
| **112** | 1 | 1 | 2 | 已下架 | 已提交，已审核，已下架（被封禁） |

### 2.3 状态流转图

```
                    ┌─────────────────────────────────────┐
                    │                                     │
                    ▼                                     │
┌───────┐  提交审核  ┌───────┐  审核通过  ┌───────┐       │
│  000  │ ────────► │  100  │ ────────► │  111  │       │
│ 草稿  │           │ 待审核 │           │ 已发布 │       │
└───────┘           └───────┘           └───────┘       │
    ▲                   │                     │         │
    │                   │  审核退回           │ 封禁    │
    │                   ▼                     └──► ┌───────┐
    │              ┌───────┘                        │  112  │
    └──────────────│  000  │                        │已下架 │
                   │ 草稿  │                        └───────┘
                   └───────┘
```

### 2.4 superAdmin 审核操作

| 审核选项 | 操作 | 结果状态 | 通知内容 |
|----------|------|----------|----------|
| 1. 【管理员评估后的】内容质量太差or【机构】申请下架 | 选择「不再受理」 | 112（已下架） | 该课程已永久下架，不再接受审核 |
| 2. 退回修改 | 允许重新提交 | 000（草稿） | 已退回，请修改后重新提交 |
| 3. 审核通过 | 直接发布 | 111（已发布） | 审核通过，课程已发布 |

## 三、站内信系统设计

### 3.1 消息类型

| 类型 | 发送方 | 接收方 | 触发场景 |
|------|--------|--------|----------|
| **COURSE_SUBMIT** | teacher | 所有 super_admin | 课程提交审核 |
| **AUDIT_APPROVED** | super_admin | 课程所属 teacher | 审核通过 |
| **AUDIT_REJECTED** | super_admin | 课程所属 teacher | 审核退回 |
| **AUDIT_BANNED** | super_admin | 课程所属 teacher | 永久下架 |

### 3.2 消息表结构建议

```sql
CREATE TABLE `system_message` (
  `id` bigint NOT NULL AUTO_INCREMENT,
  `type` varchar(32) NOT NULL COMMENT '消息类型',
  `title` varchar(200) NOT NULL COMMENT '标题',
  `content` text NOT NULL COMMENT '内容',
  `course_id` bigint DEFAULT NULL COMMENT '关联课程ID',
  `from_user_id` bigint DEFAULT NULL COMMENT '发送者ID',
  `to_user_id` bigint DEFAULT NULL COMMENT '接收者ID（单发）',
  `to_role` varchar(32) DEFAULT NULL COMMENT '接收角色（广播：super_admin）',
  `is_read` tinyint NOT NULL DEFAULT 0 COMMENT '是否已读',
  `create_time` datetime DEFAULT CURRENT_TIMESTAMP,
  PRIMARY KEY (`id`),
  KEY `idx_to_user` (`to_user_id`),
  KEY `idx_to_role` (`to_role`),
  KEY `idx_course_id` (`course_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='站内信';
```

## 四、课程访问权限矩阵

### 4.1 权限定义

| 权限项 | 说明 | 简称 |
|--------|------|------|
| **A-信息** | 课程详情、大纲、媒资列表、时长等 | info |
| **B-媒资** | 播放视频、下载文件等 | media |

### 4.2 访问权限矩阵

| 用户状态 | 免费课 | 收费课-试看章节 | 收费课-完整章节 |
|----------|--------|-----------------|-----------------|
| **未登录** | ✅ A | ✅ A | ✅ A |
| **已登录** | ✅ A + B | ✅ A + B | ✅ A |
| **已付费** | ✅ A + B | ✅ A + B | ✅ A + B |

### 4.3 权限判断逻辑

```javascript
// 伪代码
function canAccessMedia(user, course, teachplan) {
  // 1. 课程必须是已发布状态
  if (course.status !== 111) return false;

  // 2. 用户必须登录
  if (!user) return false;

  // 3. 免费课直接通过
  if (course.charge === '201000') return true;

  // 4. 收费课判断试看
  if (teachplan.is_preview_enabled === '1') return true;

  // 5. 收费课判断是否已购买
  return hasUserPurchased(user.id, course.id);
}
```
