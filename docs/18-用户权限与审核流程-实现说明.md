# 18 - 用户权限与审核流程 - 实现说明

## 一、数据库脚本执行顺序

请按以下顺序执行 SQL（若已执行过前序脚本可跳过）：

1. `initSql/xcplus_content.sql` - 课程基础表
2. `initSql/xcplus_content_media.sql` - 媒资与上传表
3. `initSql/xcplus_content_auth.sql` - 用户、购买、测试账号

## 二、MinIO 配置

- **bucket 自动创建**：启动时若配置的 bucket 不存在会自动创建（`MinioConfig.ensureBucket`）
- **分桶存储**：可选配置 `bucket-video`、`bucket-doc`，不配置则视频和文档均使用 `bucket`
- 你当前 `application-local.yml` 中 `bucket: xcplus3` 会在启动时自动创建

## 三、用户角色与权限

| 角色 | 说明 | 机构 | 权限 |
|------|------|------|------|
| super_admin | 系统管理员 | 无 | 审核所有机构课程、预览任意课程 |
| teacher | 机构教师 | company_id 必填 | 本机构课程 CRUD、提交审核、预览 |
| visitor | 访客/学员 | 无 | 浏览已发布课程、试看、购买后观看 |

## 四、课程审核流程

1. **教师**：创建/编辑课程 → 保存草稿（audit_status=202002）→ 提交审核（202003）
2. **SuperAdmin**：查看待审核列表 → 预览课程（含视频、教学文件）→ 通过（202004）或 不通过（202001）
3. **发布**：仅当 audit_status=202004 时可发布（publish_status=203002）

## 五、API 说明

### 认证

- `POST /auth/login?username=xxx&password=xxx` - 登录，返回 JWT
- 后续请求需在 Header 中携带：`Authorization: Bearer <token>`

### 教师

- `POST /courses/{id}/submit` - 提交审核
- 教师登录后可不传 `X-Company-Id`，系统从 JWT 解析机构

### 管理员

- `GET /admin/courses/pending` - 待审核课程列表
- `GET /admin/courses/{id}` - 课程详情（审核预览）
- `GET /admin/courses/{id}/teachplans` - 教学计划树（审核预览）
- `POST /admin/courses/{id}/audit?approved=true|false` - 审核通过/不通过
